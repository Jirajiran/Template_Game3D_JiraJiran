# DocCode — เรียนรู้ควบคู่ FPS Game Template (Phase 0–12)

อ้างอิงหลัก: https://learn.unity.com/
แนะนำเริ่ม: "Get Started with Unity" + "Set up the Unity Editor" + Game Development Pathway

วิธีใช้ไฟล์นี้:
1. เรียนหัวข้อ Unity Learn ในแต่ละ Phase
2. รัน FPSGame → Setup Phase X ใน Unity
3. เปิดโค้ดใน template/Scripts/ คู่กับตัวอย่างด้านล่าง
4. ทดสอบตาม template/Plan/UnitySetup.md

================================================================================
PHASE 0 — ต้องรู้: Unity กับ Vault คืออะไร
================================================================================

เป้าหมาย Phase 0
  เมื่อจบ Phase 0 คุณต้องอธิบายได้ว่า:
  - Unity project กับ vault (template) ต่างกันอย่างไร
  - GameObject / Component / Scene / Prefab คืออะไร
  - ทำไมต้อง copy Scripts แล้วกดเมนู FPSGame → Setup Phase 0
  - เปิด Unity Editor แล้วรู้ว่า Hierarchy, Inspector, Project, Scene ใช้ทำอะไร

--------------------------------------------------------------------------------
1. สองโลกที่ต้องแยกให้ออก — Vault vs Unity Project
--------------------------------------------------------------------------------

Vault (MyGameProject_Vault_base/template/)
  ที่เก็บ "ต้นฉบับ" ที่ Git / Obsidian / Cursor ดูแล
  - template/Scripts/     = โค้ด C# (ยังไม่ compile จนกว่าจะเอาเข้า Unity)
  - template/Plan/        = แผน + UnitySetup (ขั้นตอนละเอียด)
  - template/LearnCode/   = เอกสารเรียน (ไฟล์นี้)

Unity Project (เช่น FPSGame_Unity/)
  โปรเจกต์ที่ Unity Editor เปิด — มี Assets/, ProjectSettings/, Library/
  - Assets/FPSGame/Scripts/  ← copy มาจาก vault
  - Assets/FPSGame/Scenes/   ← Phase 0 สร้าง PrototypeGameplay.unity
  - Assets/FPSGame/Prefabs/  ← Phase 1+ สร้าง Player.prefab ฯลฯ

ทำไมแยกกัน?
  Unity ต้องการโครงสร้างเฉพาะ + compile + สร้าง .meta ให้ทุก asset
  Vault เก็บแค่ "source" ที่พกพาได้ — clone จาก GitHub แล้ว copy เข้า Unity ใหม่ได้

Flow มาตรฐาน (ทำครั้งแรก):
  1) Unity Hub → New project → 3D (Built-in หรือ URP)
  2) Copy  template/Scripts/  →  Assets/FPSGame/Scripts/  (รวม Editor/ และ .asmdef)
  3) รอ Unity compile (ดู Console ไม่มี error สีแดง)
  4) เมนู  FPSGame → Setup Phase 0 (Project Bootstrap)
  5) เมนู  FPSGame → Open PrototypeGameplay Scene
  6) กด Play — ยังไม่มี Player จนกว่าจะรัน Phase 1 (ปกติ)

  รายละเอียดเพิ่ม: template/UnityBootstrap/README.md
  ขั้นตอน Inspector: template/Plan/UnitySetup.md → Phase 0

2. Unity Editor — หน้าต่างที่ต้องคุ้น (ไม่ต้องจำทุกเมนู)
--------------------------------------------------------------------------------

Scene view
  มองฉาก 3D — ลากวาง object, ดูแสง, ทดสอบ Play mode

Game view
  ภาพที่ผู้เล่นจะเห็น (จากกล้อง Main Camera)

Hierarchy
  รายชื่อ GameObject ทั้งหมดในฉากปัจจุบัน
  ตัวอย่างหลัง Phase 0:
    PrototypeGameplay
    ├── Directional Light
    ├── Ground
    ├── PlayerSpawn        ← จุดเกิด (empty object)
    ├── EventSystem
    └── HUD_Canvas

Inspector
  แสดง Component ของ object ที่เลือก — แก้ค่า, ลาก reference
  ตัวอย่าง: เลือก Ground → เห็น Transform, Mesh Collider, Layer = Ground

Project
  ไฟล์ทั้งหมดใต้ Assets/ — script, scene, prefab, .asset
  โครงหลัง Phase 0:
    Assets/FPSGame/
      Scripts/   Data/   Prefabs/   Scenes/   UI/   Animation/

Console
  ข้อความ error (แดง), warning (เหลือง), Debug.Log (ขาว)
  compile ไม่ผ่าน = แก้ก่อนรัน Setup Phase ถัดไป

3. คำศัพท์หลัก — ต้องเข้าใจ ไม่ใช่แค่ท่อง
--------------------------------------------------------------------------------

GameObject
  "กล่อง" ในฉาก — ทุกอย่างใน Unity เริ่มจาก GameObject
  ตัวมันไม่ทำอะไรเอง จนกว่าจะติด Component

Component
  ความสามารถที่ติดบน GameObject
  | Component          | ทำอะไร                          |
  |--------------------|----------------------------------|
  | Transform          | ตำแหน่ง / หมุน / ขนาด           |
  | Camera             | มองฉาก                           |
  | CharacterController| เคลื่อนที่แบบ FPS (Phase 2)      |
  | CharacterBase (cs) | HP, damage — สคริปต์เราเขียน     |
  | Mesh Collider      | ชนพื้น / raycast                 |

  สคริปต์ C# ของเราก็เป็น Component หนึ่ง — แปะบน GameObject แล้ว Unity เรียกให้

Scene
  ไฟล์ .unity หนึ่งฉาก — เก็บว่ามี object อะไรอยู่ตรงไหน
  template ใช้หลาย scene หลัง Phase 10: Intro, ProfileSelection, MainMenu, PrototypeGameplay

Prefab
  แม่แบบ GameObject — แก้ prefab หนึ่งครั้ง instance ทุกตัวใน scene อัปเดตตาม
  Phase 1 สร้าง Player.prefab — Phase 8 spawn Enemy prefab

Tag
  ป้ายชื่อ string — หา object ด้วย code
  Phase 0 สร้าง: Player, Enemy
  ตัวอย่าง:
    var player = GameObject.FindGameObjectWithTag("Player");

Layer
  ชั้น 0–31 — ใช้กรอง raycast / collision (ยิงโดนอะไร, ชนอะไร)
  Phase 0 สร้าง: Ground, Player, Enemy, Hitbox
  ตัวอย่าง:
  [SerializeField] private LayerMask hitMask;   // เลือก layer ใน Inspector

4. C# ใน Unity — สิ่งที่ Phase 0 ต้องรู้ก่อน Phase 1
--------------------------------------------------------------------------------

MonoBehaviour
  คลาสที่ Unity "เรียกให้" ตาม lifecycle — สคริปต์ gameplay เกือบทั้งหมดสืบจากนี้

  public class HelloFPS : MonoBehaviour
  {
      void Awake()  { }   // ก่อน Start — ตั้งค่า reference ระหว่าง component
      void Start()  { Debug.Log("Phase 0 OK"); }  // ครั้งแรกเมื่อ object เปิด
      void Update() { }   // ทุก frame — input, logic ต่อเนื่อง
  }

  ลำนิยม: Awake → Start → Update (ทุก frame) → OnDestroy (ตอนถูกลบ)

SerializeField — ลากค่าใน Inspector ได้ แม้ field เป็น private
  [SerializeField] private float walkSpeed = 5f;
  // ใน Editor: เลือก object → Inspector → พิมพ์หรือลากค่า walkSpeed

  ทำไมไม่ public ทั้งหมด?
  Encapsulation — โค้ดภายนอกไม่ควรแก้ค่าโดยตรง; เปิดทางผ่าน method/property

Assembly Definition (.asmdef)
  แยกโค้ดเป็น assembly — compile เร็วขึ้น, แยก Runtime กับ Editor
  template มี:
    FPSGame.Runtime.asmdef   → gameplay (เข้า build เกม)
    FPSGame.Editor.asmdef    → เมนู FPSGame → Setup Phase X (Editor เท่านั้น)

  ถ้า copy Scripts แล้วไม่มี .asmdef = Unity รวมกับ Assembly-CSharp ก็ยังใช้ได้
  แต่ template ออกแบบให้มี .asmdef — copy ทั้งโฟลเดอร์ Scripts/

Debug.Log
  พิมพ์ข้อความใน Console — ใช้ตรวจว่า code ทำงานถึงจุดไหน
  Debug.Log("[FPSGame] Phase 0 setup complete.");

1. ปัญหาที่ Phase 1 แก้ — ตัวละครต้องมี "ศูนย์กลาง HP"
--------------------------------------------------------------------------------

เกม FPS มีหลาย "unit": ผู้เล่น, ศัตรู AI, dummy ทดสอบ
ทุกตัวต้อง:
  - มี HP สูงสุด / HP ปัจจุบัน
  - รับ damage, heal, ตาย, ฟื้น
  - บอก UI ว่า HP เปลี่ยน (Phase 4)

ถ้าเขียน HP ซ้ำใน Player.cs และ Enemy.cs = แก้ bug สองที่, balance ยาก

template แก้ด้วย:
  CharacterStats (SO)     = ข้อมูล balance ต่อตัวละคร (maxHealth, walkSpeed, faction)
  CharacterBase (Script)  = logic HP ร่วม (TakeDamage, Heal, Die, events)
  PlayerController        = input ผู้เล่น (Phase 2 — แยก component)
  AIController            = FSM ศัตรู (Phase 7 — แยก component)

Player ไม่ได้ "สืบทอด" CharacterBase แบบ class Player : CharacterBase
แต่ใช้ composition — หลาย component บน GameObject เดียว (ดูข้อ 4)

2. ScriptableObject (SO) — ข้อมูลที่อยู่ใน Project ไม่ใช่ในฉาก
--------------------------------------------------------------------------------

MonoBehaviour  = ติดบน GameObject ในฉาก (Player ใน scene)
ScriptableObject = ไฟล์ .asset ใน Project window — ไม่มีตำแหน่งในโลก 3D

ทำไมใช้ SO สำหรับ stats?
  - นัก design แก้ maxHealth ใน Inspector โดยไม่แตะโค้ด
  - ฮีโร่หลายตัว = หลาย .asset (Hero_Default, Hero_Tank, ...)
  - Save/loadout อ้าง characterId string แล้ว lookup SO (Phase 9)

CharacterStats ใน template:

  [CreateAssetMenu(fileName = "CharacterStats", menuName = "FPSGame/Character Stats")]
  public class CharacterStats : ScriptableObject
  {
      public string characterId;      // "hero_default" — ใช้ใน JSON save
      public float maxHealth = 100f;
      public float walkSpeed = 5f;
      public float sprintSpeed = 8f;
      public float jumpHeight = 1.5f;
      public int jumpLimit = 1;
      public Faction faction = Faction.Friendly;   // Friendly | Enemy
  }

Phase 1 สร้างไฟล์:
  Assets/FPSGame/Data/Characters/Hero_Default.asset
  characterId = hero_default, maxHealth = 100, faction = Friendly

สร้างมือ (ถ้าไม่ใช้เมนู):
  Project → คลิกขวา → Create → FPSGame → Character Stats

--------------------------------------------------------------------------------
3. CharacterBase — logic HP ที่ Player และ AI ใช้ร่วมกัน
--------------------------------------------------------------------------------

ไฟล์: template/Scripts/Core/CharacterBase.cs

อ่าน stats จาก SO ที่ลากใน Inspector:

  [SerializeField] private CharacterStats stats;

  public float MaxHealth => stats != null ? stats.maxHealth : 0f;
  public float CurrentHealth => currentHealth;
  public bool IsAlive => !isDead;
  public Faction Faction => stats != null ? stats.faction : Faction.Friendly;

Runtime HP เก็บใน private field (ไม่แก้ SO ตอนเล่น):

  private float currentHealth;        // HP จริง (Primary bar — Phase 4)
  private float displayDelayHealth;   // HP แถบ delay ค่อยๆ ตาม (visual)

Awake — ตั้ง HP เริ่มจาก stats:

  void Awake()
  {
      if (stats == null) { Debug.LogError("CharacterStats not assigned"); return; }
      ResetHealthState();   // currentHealth = maxHealth
  }

TakeDamage — ลด HP, แจ้ง event, ตายถ้า 0:

  public virtual void TakeDamage(float amount)
  {
      if (!IsAlive || amount <= 0f) return;
      float previous = currentHealth;
      currentHealth = Mathf.Max(0f, currentHealth - amount);
      OnHealthChanged?.Invoke(previous, currentHealth);
      if (currentHealth <= 0f) Die();
  }

Heal / Revive / Die — ชุด API ที่ UI และ gameplay เรียกผ่าน public method

ApplyStats — เปลี่ยนฮีโร่ runtime (loadout / save Phase 9):

  public void ApplyStats(CharacterStats newStats)
  {
      stats = newStats;
      ResetHealthState();
  }

virtual บน TakeDamage, Heal, Die
  คลาสลูก (ถ้ามีในอนาคต) override ได้ — ตามหลัก OOP ใน Plantext

Phase 6+: CharacterBase implements IDamageable — อาวุธเรียก TakeDamage ผ่าน interface

--------------------------------------------------------------------------------
4. Composition — ทำไมไม่ใช้ class Player : CharacterBase
--------------------------------------------------------------------------------

Plantext เก่าอธิบายแบบ TF2: UnitBase → PlayerUnit / EnemyUnit (inheritance)

template จริงใช้ composition (architecture lock):

  GameObject "Player"  (Tag: Player, Layer: Player)
  ├── CharacterBase           ← HP, damage, faction
  ├── CharacterController     ← เคลื่อนที่ (Phase 2)
  ├── PlayerController        ← input WASD (Phase 2)
  ├── CharacterDamageTester   ← K/H/Y ทดสอบ (prototype)
  ├── Camera                  ← มุมมอง FPS
  └── WeaponMount             ← WeaponHandler (Phase 3)

เหตุผล:
  - Player รับ input จากคนดเล่น — AI รับคำสั่งจาก FSM
  - logic HP เหมือนกัน → อยู่ CharacterBase
  - logic ควบคุมต่างกัน → component คนละตัว (PlayerController vs AIController)
  - ไม่ต้องสร้าง inheritance tree ลึก — ขยายด้วย component ใหม่ได้

PlayerController (Phase 2) จะ GetComponent<CharacterBase>() อ่าน HP / IsAlive
ไม่ copy logic HP ซ้ำใน PlayerController

5. C# Event — ส่งสัญญาณโดยไม่รู้ว่าใครฟัง (Loose Coupling)
--------------------------------------------------------------------------------

CharacterBase ประกาศ event:

  public event Action<float, float> OnHealthChanged;   // (previous, current)
  public event Action<float> OnDelayHealthChanged;
  public event Action<float> OnHealed;
  public event Action OnDied;
  public event Action OnRevived;

เมื่อ HP เปลี่ยน:

  OnHealthChanged?.Invoke(previous, currentHealth);
  // ?. = เรียกเฉพาะเมื่อมีคน subscribe — ไม่ error ถ้ายังไม่มี UI

Phase 4 HUD จะ subscribe:

  void OnEnable()  { character.OnHealthChanged += UpdateHealthBar; }
  void OnDisable() { character.OnHealthChanged -= UpdateHealthBar; }

หลักสำคัญ:
  CharacterBase ไม่รู้จัก HUD class
  HUD ไม่ควรเรียก TakeDamage เพื่อ "ทดสอบ bar"
  → แยกชั้นชัด: gameplay เปลี่ยน state → event → UI แสดงผล

6. Dual HP bar (logic ใน Phase 1, UI ใน Phase 4)
--------------------------------------------------------------------------------

Primary HP (currentHealth)
  ลดทันทีเมื่อ TakeDamage — ค่าจริงของ gameplay

Delay HP (displayDelayHealth)
  ใน Update: ค่อยๆ MoveTowards ไปหา currentHealth
  OnDelayHealthChanged แจ้ง UI แถบหลัง

  private void TickDelayHealthBar()
  {
      displayDelayHealth = Mathf.MoveTowards(
          displayDelayHealth, currentHealth,
          delayBarCatchUpPerSecond * Time.deltaTime);
  }

Phase 1: ดูใน Inspector ตอนกด K — CurrentHealth ลดทันที, DisplayDelayHealth ตามทีหลัง
Phase 4: DualLayerHealthBarUI แสดงสองแถบ

8. CharacterDamageTester — ทดสอบใน Play mode (prototype เท่านั้น)
--------------------------------------------------------------------------------

ไฟล์: Core/CharacterDamageTester.cs
ติดบน Player — ห้ามใส่ใน build จริง (ลบ/disable ก่อนปล่อยเกม)

  if (Input.GetKeyDown(KeyCode.K)) character.TakeDamage(10f);
  if (Input.GetKeyDown(KeyCode.H)) character.Heal(10f);
  if (Input.GetKeyDown(KeyCode.Y) && !character.IsAlive) character.Revive();

K = damage 10 | H = heal 10 | Y = revive เมื่อตาย

ดูผล:
  Inspector → เลือก Player → CharacterBase → Current Health ลด/เพิ่ม

2. Legacy Input Manager — template ใช้ตัวนี้ (ไม่ใช่ Input System package)
--------------------------------------------------------------------------------

Unity มี 2 ระบบ input — template นี้ใช้ Legacy (Built-in):

  Project Settings → Input Manager (หรือ axes เริ่มต้นของ Unity)

  แกน / ปุ่มที่ PlayerController ใช้:

  | Input API                    | ปุ่ม / แกน        | ทำอะไร              |
  |------------------------------|-------------------|----------------------|
  | GetAxisRaw("Horizontal")     | A / D             | ซ้าย-ขวา             |
  | GetAxisRaw("Vertical")       | W / S             | หน้า-หลัง            |
  | GetButtonDown("Jump")        | Space             | กระโดด               |
  | GetAxis("Mouse X/Y")         | ขยับเมาส์         | มองรอบ               |
  | GetMouseButton(0)            | LMB               | ยิง / ฟัน (primary)  |
  | GetMouseButton(1)            | RMB               | aim (เดินช้า)        |
  | GetKey(LeftShift/RightShift) | Shift             | sprint               |
  | GetKeyDown(R)                | R                 | reload               |
  | GetKeyDown(Alpha1/2/3)       | 1 / 2 / 3         | weapon slot          |
  | GetAxis("Mouse ScrollWheel") | scroll            | สลับ slot            |

GetAxisRaw vs GetAxis:
  Raw = -1, 0, 1 ทันที (WASD) — เดิน FPS ใช้ Raw
  Axis มี smoothing — mouse ใช้ GetAxis("Mouse X") ได้

ไม่ต้องติดตั้ง com.unity.inputsystem ถ้าทำตาม template นี้
(UnityBootstrap อาจพูดถึง Input System — ข้ามได้ถ้าใช้ Legacy)

3. CharacterController — เคลื่อนที่ FPS แบบไม่ใช้ Rigidbody
--------------------------------------------------------------------------------

Rigidbody + Collider = ฟิสิกส์จริง (กระเด็น, ถูก force ผลัก)
CharacterController = capsule เดินเอง — เหมาะ FPS / third-person ทั่วไป

Phase 2 Setup ลบ Rigidbody + CapsuleCollider เก่า (ถ้ามีจาก Phase 1 รุ่นแรก)
แล้วใส่ CharacterController:

  height = 2, radius = 0.4, center = (0, 1, 0)
  slopeLimit = 45, stepOffset = 0.3

เคลื่อนที่ด้วย Move() — ไม่ใช้ transform.position ตรงๆ:

  controller.Move(moveDirection * speed * Time.deltaTime);

isGrounded — เช็คติดพื้นก่อนกระโดด:

  bool grounded = controller.isGrounded;
  character.SetGrounded(grounded);   // ส่งให้ CharacterBase รู้ (animation ภายหลัง)

Gravity + Jump ใช้ velocity.y แยกจากการเดินแนวราบ:

  if (Input.GetButtonDown("Jump") && grounded && jumpsUsed < stats.jumpLimit)
      velocity.y = Mathf.Sqrt(stats.jumpHeight * -2f * stats.gravity);

  velocity.y += stats.gravity * Time.deltaTime;
  controller.Move(velocity * Time.deltaTime);

ค่า jumpHeight, gravity, jumpLimit มาจาก CharacterStats (SO) — ไม่ hardcode ใน PlayerController

Ground ใน scene ต้องมี Collider + Layer Ground (Phase 0)

--------------------------------------------------------------------------------
4. FPS Look — แยก Yaw (ตัว) กับ Pitch (กล้อง)
--------------------------------------------------------------------------------

FPS มาตรฐาน:
  หมุนซ้าย-ขวา (Yaw)   → หมุน root Player (transform)
  หมุนขึ้น-ลง (Pitch)  → หมุน Camera child เท่านั้น

  // Yaw — โลก
  transform.Rotate(Vector3.up, mouseX, Space.World);

  // Pitch — local กล้อง, clamp ไม่ให้หมุนคว่ำ
  pitch -= mouseY;
  pitch = Mathf.Clamp(pitch, minPitch, maxPitch);   // default -80 .. 80
  cameraPivot.localRotation = Quaternion.Euler(pitch, 0f, 0f);

cameraPivot = Transform ของ Camera (ลากใน Inspector)
ทิศเดิน WASD ใช้ transform.forward / transform.right ของ Player root
→ เดินตามทิศที่หันหน้า (yaw) ไม่ตาม pitch

Cursor lock ตอนเล่น:

  Cursor.lockState = CursorLockMode.Locked;
  Cursor.visible = false;
  // GameplayPauseUI / Phase 5 จะปลด lock ตอน pause menu

5. PlayerController — โครงสร้างและ RequireComponent
--------------------------------------------------------------------------------

ไฟล์: template/Scripts/Core/PlayerController.cs

  [RequireComponent(typeof(CharacterBase))]
  [RequireComponent(typeof(CharacterController))]
  public class PlayerController : MonoBehaviour

RequireComponent = Unity บังคับให้มี component คู่บน GameObject เดียวกัน

Reference ใน Inspector:

  [SerializeField] private Transform cameraPivot;      // Camera transform
  [SerializeField] private WeaponHandler weaponHandler; // บน WeaponMount (Phase 3)
  [SerializeField] private float mouseSensitivity = 2f;
  [SerializeField] private float aimMoveSpeedMultiplier = 0.5f;

Awake — หา reference อัตโนมัติถ้าไม่ได้ลาก:

  character = GetComponent<CharacterBase>();
  controller = GetComponent<CharacterController>();
  weaponHandler = GetComponentInChildren<WeaponHandler>();

Update loop (ลำดับสำคัญ):

  if (GameplayPauseService.IsPaused) return;     // Phase 5
  if (!character.IsAlive) return;               // ตายแล้วไม่รับ input

  HandleLook();
  HandleMovement();
  HandleWeaponInput();
  UpdateAnimator();   // optional — MoveX, MoveY, Speed

6. การเดิน — อ่าน stats, sprint, aim slow
--------------------------------------------------------------------------------

ทิศจาก input สัมพันธ์กับทิศหันหน้า:

  Vector3 move = transform.right * horizontal + transform.forward * vertical;
  if (move.sqrMagnitude > 1f) move.Normalize();   // ไม่วิ่งเร็วขึ้นเมื่อกด W+D

ความเร็วจาก CharacterStats:

  float speed = stats.walkSpeed;
  if (isAiming)       speed *= aimMoveSpeedMultiplier;   // RMB ค้าง — ช้าลง
  else if (isSprinting) speed = stats.sprintSpeed;     // Shift + WASD

Sprint ต้องกำลังเคลื่อนที่ และไม่ aim:

  isSprinting = sprint && (horizontal != 0 || vertical != 0) && !isAiming;

  เรียกจาก PlayerController ตอน sprint — ไม่ต้องมี component บน Player ก็ทำงาน (static)

2. ลำดับชั้น WeaponData — SO hierarchy
--------------------------------------------------------------------------------

  WeaponData (abstract SO)
  ├── RangedWeaponData (abstract)
  │   ├── HitscanWeaponData    ← sniper (ทันที, แม่น, ไกล)
  │   └── ProjectileWeaponData ← pistol, AK (RaycastProjectile — หลบได้)
  └── MeleeWeaponData          ← knife

WeaponData — ค่าร่วมทุชนิด:

  public string weaponId;        // "pistol_01" — ใช้ใน JSON save
  public string displayName;     // "Pistol" — แสดง HUD
  public WeaponCategory category; // Hitscan | Projectile | Melee
  public float damage;
  public float attackCooldown;   // ระยะห่างระหว่าง shot/swing

RangedWeaponData — เฉพาะปืน:

  public int clipSize = 30;
  public int magazineCount = 3;      // แมกสำรอง (reserve)
  public float spreadDegrees = 1f;
  public float reloadPhaseDuration = 0.2f;   // ต่อ phase

ProjectileWeaponData — ปืนกระสุนบิน (RaycastProjectile):

  public GameObject projectilePrefab;
  public float projectileSpeed = 90f;   // หลบได้บางระยะ
  public float projectileGravity = 0f;
  public float projectileMaxLifetime = 3f;

HitscanWeaponData — ปืน ray ทันที (ทางเลือก):

  public float baseHitDelay = 0.01f;
  public float extraDelayPer100m = 0.01f;
  public float maxRange = 300f;
  public int penetration = 0;        // ทะลุ unit ได้กี่ตัว (0 = 1 เป้า)
  public GameObject bulletHolePrefab;

MeleeWeaponData — มีด/ค้อน:

  public float activeTime = 0.01f;   // hitbox อยู่กี่วินาที
  public Vector3 hitboxSize;
  public Vector3 hitboxOffset;       // 相对 muzzle

| AK | 25 | 0.10s | 30 | speed 90, g=0 | damage/speed เหมือน pistol, ยิงถี่กว่า |
| Sniper | 85 | 1.2s | 5 | maxRange 500m | hitscan, spread ต่ำ |

ระยะเวลาหลบ (projectile): ประมาณ ระยะ(m) ÷ speed — เช่น 18m ÷ 90 ≈ 0.2s

--------------------------------------------------------------------------------
2c. Hitscan vs Projectile — เลือกเมื่อไหร่
--------------------------------------------------------------------------------

Hitscan (Sniper):
  - Physics.Raycast ครั้งเดียวจาก muzzle — โดนทันทีที่อยู่ในเล็ง
  - เหมาะปืนเร็วมาก / แม่น / ไกล — ผู้เล่นหลบยาก
  - delay เล็กน้อยจำลองระยะ (ScheduleHitscanDamage) ไม่ใช่กระสุนบิน

Projectile + RaycastProjectile (Pistol, AK):
  - Instantiate กระสุน → บินทีละเฟรม → Raycast ตาม segment กันทะลุ
  - speed + gravity กำหนดเวลาถึงเป้า — **หลบด้วย strafe/sprint ได้**
  - damage ผ่าน HitRegistry.TryApplyDamage เมื่อชน unit

Melee (Knife):
  - ไม่มี ray / กระสุน — trigger hitbox สั้นๆ

กฎออกแบบ template:
  - ปืนสั้น/กลางที่อยากให้มี skill expression → Projectile
  - ปืนพิเศษทันที → Hitscan
  - ระยะสั้น ≠ ต้อง hitscan — pistol ใช้ projectile แม้ระยะใกล้

4. TryPrimaryAction — แยกตาม WeaponCategory
--------------------------------------------------------------------------------

  public void TryPrimaryAction()
  {
      if (IsReloading) CancelReload();     // LMB ระหว่าง reload = ยกเลิก
      if (Time.time < nextFireTime) return;

      switch (weapon.category)
      {
          case WeaponCategory.Hitscan:
              if (TryFireRanged(...)) FireHitscan(...);
              break;
          case WeaponCategory.Melee:
              StartMelee(...);
              break;
          case WeaponCategory.Projectile:
              if (TryFireRanged(...)) FireProjectile(...);
              break;
      }
  }

TryFireRanged — ก่อนยิงจริง:
  - clipAmmo <= 0 → TryReload() อัตโนมัติ, return false
  - clipAmmo-- , ตั้ง nextFireTime จาก attackCooldown

Melee — ไม่ใช้ clip (ไม่มี TryFireRanged)

--------------------------------------------------------------------------------
5. Hitscan — ray จาก muzzle, delay damage, penetration
--------------------------------------------------------------------------------

FireHitscan ขั้นตอน:

  1) origin = muzzle.position, direction = muzzle.forward (+ spread)
  2) HitRegistry.ResolveHitscanRay(...) → รายการ unit ที่โดน + obstacle
  3) แต่ละ unit: คำนวณ delay ตามระยะ → ScheduleHitscanDamage
  4) ชนสิ่งกีดขวาง (ไม่ใช่ unit): SpawnBulletHole prefab

Delay (Plantext §4):
  ใกล้ = ~0.01s, ไกล = baseHitDelay + extra ต่อ 100m เกิน referenceRange

Penetration:
  penetration = 0 → โดน hostile unit ได้ 1 ตัว (count = penetration + 1)
  friendly / dead ข้าม — ไม่กิน penetration slot

  Phase 3 ทดสอบ: ยิง TestDummy @ (5,1,0) — HP ลดหลัง delay เล็กน้อย
  ยิงผนัง/Ground — bullet_hole_placeholder ปรากฏ

Raycast แนวคิด (HitRegistry ทำให้):

  Physics.Raycast(origin, direction, out hit, maxRange, hitMask);

Phase 6 polish HitRegistry + IDamageable — Phase 3 ใช้ path เดียวกันแล้ว

Sniper ทดสอบ: ใส่ใน slot (loadout) → ยิง dummy — โดนทันที + delay เล็กน้อย, damage สูง

--------------------------------------------------------------------------------
5b. Raycast Projectile — Pistol, AK (FireProjectile)
--------------------------------------------------------------------------------

ไฟล์: Weapons/RaycastProjectile.cs (บน RaycastBullet.prefab)

FireProjectile (WeaponHandler):
  Instantiate(projectilePrefab) ที่ muzzle
  RaycastProjectile.Launch(direction, speed, damage, owner, hitMask, gravity, lifetime)

แต่ละเฟรม (Update):
  velocity += gravity ลง (AK/pistol = 0 → ไม่โค้ง)
  Raycast ตาม distanceThisFrame → ชน → impact VFX + HitRegistry.TryApplyDamage
  ไม่ชน → ขยับตำแหน่ง + หมุนตาม velocity

Pistol vs AK (ค่าเดียวกัน damage/speed):
  AK ยิงถี่กว่า (attackCooldown 0.1 vs 0.25), แมกใหญ่กว่า (30 vs 12)

ทดสอบหลบ: ให้ศัตรู/ผู้เล่นยิง AK ขณะคุณ strafe — บางนัดหลบได้ถ้าระยะพอ

RPG: gravity 9.81, speed 80 — โค้งลง, ช้ากว่า pistol (unlock ใน StartPack)

6. Melee — hitbox ชั่วคราว, 1 hit ต่อเป้า ต่อ swing
--------------------------------------------------------------------------------

StartMelee:

  1) meleeSwingSequence++ → swingId ไม่ซ้ำต่อ swing
  2) สร้าง GameObject ชั่วคราว + BoxCollider (isTrigger)
  3) Layer Hitbox (Phase 0)
  4) MeleeHitbox.Initialize(owner, damage, size, swingId)
  5) Destroy(hitboxGo, activeTime);   // ~0.01s

MeleeHitbox.OnTriggerEnter:

  HitRegistry.TryGetDamageable(collider, out target)
  HitRegistry.TryApplyMeleeHit(swingId, target, damage, owner)
  → กันตีซ้ำ target เดิมใน swing เดียว

ทดสอบ: slot 2 (knife) + LMB ใกล้ TestDummy — โดน 1 ครั้ง/swing

--------------------------------------------------------------------------------
7. Reload — 3 phase, dump clip, cancel ได้
--------------------------------------------------------------------------------

ReloadPhase enum:

  None → Eject → Insert → Chamber → None (เสร็จ)

BeginReload (กด R หรือ auto เมื่อ clip หมด):
  - clipAmmo = 0 ทันที (dump กระสุนในแมก — design template)
  - reserveMagazines > 0 และ clip ไม่เต็มอยู่แล้ว
  - แต่ละ phase ใช้เวลา reloadPhaseDuration (default 0.2s) → รวม ~0.6s

CompleteReload:
  clipAmmo = clipSize
  reserveMagazines -= 1

CancelReload:
  LMB ระหว่าง reload → หยุดกลางคัน (TryPrimaryAction เรียก CancelReload ก่อน)

TickReload ใน Update — ใช้ Time.time (หยุดเมื่อ pause Phase 5)

R กับ melee slot — TryReload return false (ไม่ใช่ RangedWeaponData)

4. Hitscan — ResolveHitscanRay + delay + penetration
--------------------------------------------------------------------------------

WeaponHandler.FireHitscan:

  1) HitRegistry.ResolveHitscanRay(origin, dir, data, owner, hitMask, buffer, ...)
  2) แต่ละ unit ใน buffer → delay = CalculateHitscanDelay(data, distance)
  3) ScheduleHitscanDamage → รอ Time.time + delay
  4) TickPendingHits → HitRegistry.TryApplyDamage(target, damage, owner)

CalculateHitscanDelay (Plantext §4):

  delay = baseHitDelay;
  if (distance > referenceRangeMeters)
      delay += ((distance - reference) / 100f) * extraDelayPer100m;

ResolveHitscanRay — Physics.RaycastAll เรียงตาม distance:

  unitsRemaining = penetration + 1    // penetration=0 → โดน 1 unit

  แต่ละ hit:
    - collider ของตัวยิง (owner child) → ข้าม
    - IDamageable hostile + alive → ใส่ unitHits, unitsRemaining--
    - friendly / dead → ข้าม (ไม่กิน penetration)
    - non-trigger obstacle (ไม่ใช่ unit) → bullet hole, หยุด ray

  penetration=0: โดน hostile unit ตัวแรกแล้ว return (ไม่ทะลุ)
  penetration=1: ทะลุได้ 2 units สูงสุด

TryApplyDamage — ชั้นสุดท้ายก่อน TakeDamage:

  if (!target.IsAlive) return false;
  if (instigator hostile check fail) return false;   // ไม่ยิงเพื่อน
  target.TakeDamage(amount);

3. FSM — AiState Patrol, Chase, Attack
--------------------------------------------------------------------------------

ไฟล์: AI/AiState.cs — enum 3 state

AIController.Update (หลัง perception.Tick):

  switch (state)
  {
      case Patrol:  TickPatrol();  break;
      case Chase:   TickChase();   break;
      case Attack:  TickAttack();  break;
  }

Patrol:
  - วน patrolPoints[] — SetDestination → รอ patrolWaitSeconds → จุดถัดไป
  - HasPerceivedTarget() → TransitionTo(Chase)

Chase:
  - SetDestination(perception.CurrentTarget.position)
  - ระยะ <= GetPreferredAttackRange() → Attack
  - ไม่มี target → กลับ Patrol

Attack:
  - agent.isStopped = true, FaceTarget (หมุนหา target)
  - SelectWeaponForDistance — ใกล้ = melee slot, ไกล = ranged slot
  - weaponHandler.TryPrimaryAction() — ยิง/ฟัน (HitRegistry path)
  - ไกลเกิน attackRange * 1.15 → Chase

TransitionTo — reset patrolWaitTimer, เปลี่ยน state

OnDied (CharacterBase.OnDied):
  agent.isStopped = true; enabled = false;

ตัวอย่างจาก AIController.cs:

  private void TickChase()
  {
      agent.isStopped = false;
      if (!HasPerceivedTarget()) { TransitionTo(AiState.Patrol); return; }
      var target = perception.CurrentTarget;
      if (Vector3.Distance(transform.position, target.transform.position)
          <= GetPreferredAttackRange())
      {
          TransitionTo(AiState.Attack);
          return;
      }
      agent.SetDestination(target.transform.position);
  }

Pause gate (Phase 5):

  if (character == null || !character.IsAlive || GameplayPauseService.IsPaused)
      return;

4. AiPerception — Sight, Hearing, Memory
--------------------------------------------------------------------------------

ไฟล์: AI/AiPerception.cs + AiPerceptionConfig.asset (SO)

Config ตัวอย่าง (EnemyPerception_Default):
  sightRange = 25, fieldOfViewDegrees = 110
  hearingRange = 18, gunshotMemorySeconds = 1.5
  loseTargetDelay = 2.5
  lineOfSightMask — layer ที่ ray LOS ชนได้

Tick() ทุก frame (จาก AIController):

  TryAcquireTarget → ตั้ง currentTarget
  ถ้ามี target แต่ไม่เห็น/ไม่ได้ยินนาน → lostTargetTimer → ClearTarget

CanSee (Sight):
  1) owner.IsHostileTo(target)
  2) ระยะ <= sightRange
  3) มุมใน FOV (horizontal — flatForward vs flatToTarget)
  4) Physics.Raycast จาก eyePoint — โดน collider ของ target จริง (ไม่มีกำแพงบัง)

CanHear (Hearing):
  1) hostile + ระยะ <= hearingRange
  2) PlayerNoiseEmitter.IsSprinting(player) — Shift วิ่ง
  3) หรือ Time.time - GetLastGunshotTime(player) <= gunshotMemorySeconds

PlayerNoiseEmitter (บน Player — Phase 7 Setup เพิ่ม):
  PlayerController sprint → SetSprinting(true/false)
  WeaponHandler ยิง → RegisterGunshot(player.gameObject)

หา Player: GameObject.FindGameObjectWithTag("Player") + CharacterBase

ตัวอย่าง CanSee (ย่อ):

  float angle = Vector3.Angle(flatForward, flatToTarget);
  if (angle > halfFov) return false;
  if (Physics.Raycast(origin, toTarget.normalized, out hit, distance, mask))
      return hit.collider.GetComponentInParent<CharacterBase>() == target;

3. EnemySpawnTrigger — Trigger volume
--------------------------------------------------------------------------------

[RequireComponent(typeof(Collider))] — Awake บังคับ col.isTrigger = true

OnTriggerEnter(Collider other):
  1) triggerOnce && hasTriggered → return (spawn ครั้งเดียว)
  2) !other.CompareTag("Player") → return
  3) hasTriggered = true → ActivateSpawnPoints()

ActivateSpawnPoints — loop spawnPoints[] → BeginSpawn() แต่ละจุด

Phase 8 scene: SpawnTrigger_01 @ (4, 1, 4), BoxCollider 8×3×8
Gizmo สีส้มใน Scene view — โซนที่ต้องเดินเข้า

ตัวอย่าง:

  private void OnTriggerEnter(Collider other)
  {
      if (triggerOnce && hasTriggered) return;
      if (!other.CompareTag("Player")) return;
      hasTriggered = true;
      ActivateSpawnPoints();
  }

Player ต้องมี Tag "Player" + Collider (CharacterController นับ)

5. EnemySpawnPoint — Coroutine spawn
--------------------------------------------------------------------------------

BeginSpawn():
  - sequence null / steps ว่าง → warning, return
  - runOncePerSpawnPoint && hasRun → return
  - isSpawning → return (กันเรียกซ้อน)
  - StartCoroutine(SpawnRoutine())

SpawnRoutine (สรุปลำดับ):
  for แต่ละ step i:
    if i > 0 && delayAfterPrevious > 0 → WaitForSeconds(delay)
    for c = 0 .. count-1:
      SpawnEnemy(prefab, patrolPoints)
      ระหว่าง count ใน step เดียว → ห่าง 0.25s

  hasRun = true; isSpawning = false

SpawnEnemy:
  Instantiate(prefab, spawnPos, spawnRot)
  NavMeshAgent → NavMesh.SamplePosition + Warp (radius 3) — ให้อยู่บน NavMesh
  AIController.ConfigurePatrol(points) — วน patrol รอบจุดลูก

Patrol points — ResolvePatrolPoints():
  default useChildPatrolPoints = true
  หา child ชื่อขึ้นต้น "PatrolPoint" (PatrolPoint_A, PatrolPoint_B)
  EnemySpawnPoint prefab มีลูก 2 จุด — ศัตรูที่ spawn วนรอบ spawn point นั้น

ConfigurePatrol (AIController):

  public void ConfigurePatrol(Transform[] points)
  {
      patrolPoints = points;
      patrolIndex = 0;
      if (agent.isOnNavMesh && points.Length > 0)
          agent.SetDestination(points[0].position);
  }

Spawn ที่ (8, 0, 8) — ไกลจาก trigger (4, 1, 4) เล็กน้อย ศัตรู patrol ก่อนเห็นผู้เล่น

2. JSON + JsonUtility — ทำไมไม่ใช้ PlayerPrefs
--------------------------------------------------------------------------------

PlayerPrefs: key-value ง่าย แต่ไม่เหมาะ list ซับซ้อน (unlock หลาย id)
Template: ไฟล์ `profile_{1..3}.json` ใต้ `Application.persistentDataPath`

  string json = JsonUtility.ToJson(Data, true);
  File.WriteAllText(path, json);
  Data = JsonUtility.FromJson<ProfileSaveData>(json);

ข้อจำกัด JsonUtility (ต้องรู้):
  - รองรับ [Serializable] class / struct — ไม่รองรับ Dictionary
  - array ของ string ใช้ได้ (unlockedWeaponIds)
  - field ต้อง public หรือ [SerializeField]

Path ตัวอย่าง (Windows):
  `%USERPROFILE%\AppData\LocalLow\<Company>\<Product>\profile_1.json`

JSON ไม่ commit ใน Git — สร้าง runtime; ทดสอบด้วย Play + F-keys

3. ProfileSaveData — โมเดลใน JSON

  public class ProfileSaveData
  {
      public int saveVersion = 1;
      public int wallet = 100;

4. Drag & Drop — Unity EventSystem interfaces
--------------------------------------------------------------------------------

LoadoutDragHandle (IBeginDragHandler, IDragHandler, IEndDragHandler):

  OnBeginDrag:
    LoadoutDragSession.BeginWeapon(id) หรือ BeginEquippedWeapon(id, slot)
    source alpha 0.55, blocksRaycasts = false
    สร้าง dragGhost ตามเมาส์
    ปิด GraphicRaycaster ชั่วคราว (ให้ drop ผ่าน ghost)

  OnDrag: ย้าย ghost ตาม ScreenPointToLocalPointInRectangle

  OnEndDrag: คืน alpha, ลบ ghost, LoadoutDragSession.Clear()

LoadoutDropTarget (IDropHandler):

  OnDrop → panel.HandleDrop(zone, slotIndex)

LoadoutDragSession — static payload (ไม่ serialize):

  Kind: Weapon | EquippedWeapon
  ItemId: weaponId
  SourceSlot: -1 (จาก list) หรือ slot index (จาก equipped)

  IsActive → Kind != None

ทำไมใช้ static session:
  drag ghost ไม่ carry reference ไป drop target — EventSystem แยก object
  pattern ง่ายสำหรับ prototype (Phase 11 ไม่ใช้ ScriptableObject drag payload)

5. HandleDrop + AssignWeaponToSlot — logic บันทึก
--------------------------------------------------------------------------------

HandleDrop ตรวจ:
  HasActiveProfile + LoadoutDragSession.IsActive
  weapon unlock แล้ว

WeaponSlot zone:
  ลากจาก slot เดิมมาวาง slot เดิม → return (no-op)
  AssignWeaponToSlot(targetSlot, weaponId, sourceSlot)

UnequipPool zone:
  ต้อง drag จาก EquippedWeapon เท่านั้น
  ClearSlot(sourceSlot)

AssignWeaponToSlot (สำคัญ):

  1) ลบ weaponId ซ้ำจาก slot อื่น (อาวุธเดียวกันได้ slot เดียว)
  2) SetLoadoutWeapon(targetSlot, weaponId, saveImmediately: false)
  3) ถ้า drag จาก slot อื่น → swap displaced weapon กลับ source slot
  4) SaveProfileService.Save() ครั้งเดียวท้าย

SelectHero:
  SetSelectedHero(heroId) — default saveImmediately=true ใน Service

ClearSlot:
  SetLoadoutWeapon(slotIndex, "") → Save()

ตัวอย่าง JSON หลังลาก knife ไป slot 2:
  "loadoutWeaponIds": ["pistol_01", "", "knife_01"]

3. Unlock rules — SaveProfileService API
--------------------------------------------------------------------------------

UnlockLevel(key) — เพิ่ม key ใน unlockedLevelKeys + Save()

UnlockNextLevelAfterWin(campaign, completedLevel):
  1) Unlock ด่านที่ชนะ (completedLevel)
  2) Unlock ด่านถัดไปใน campaign เดียวกัน
  3) ถ้าจบ campaign (level 4) → unlock c{n+1}_l1 campaign ถัดไป

  // ตัวอย่าง: ชนะ Campaign 1 Level 1
  UnlockNextLevelAfterWin(1, 1);
  // → unlock c1_l1 + c1_l2

UnlockSecretStage():
  secretStageUnlocked = true
  Unlock secret_c1 … secret_c4 (ทุก campaign)
  Save()

หมายเหตุ prototype:
  ชนะด่าน → CampaignLevelWinController → UnlockNextLevelAfterWin → JSON unlockedLevelKeys
  เงื่อนไขชนะ: ฆ่าศัตรูหมด + spawn wave จบ (Phase 8)
  Secret stage: ชนะแล้วไม่ unlock ด่านถัดไป (แค่แสดง Mission Complete)
  Debug: F7 บังคับชนะใน gameplay | F6 → UnlockSecretStage()

5. CampaignSessionState + MenuFlowState — restore หลัง pause
--------------------------------------------------------------------------------

CampaignSessionState (static — จำ context ตอนเล่น):

  BeginCampaignLevel → ActiveCampaign, ActiveLevelKey, IsSecretStage=false
  BeginSecretStage → ActiveCampaign, ActiveLevelKey=secret_c{n}
  HasActiveCampaignContext → ActiveCampaign >= 1
  Clear() เมื่อกลับ campaign list

  RequestCampaignLevelList(campaign) — หลัง launch level / pause → menu
  HasPendingCampaignRestore → CampaignPanel restore level list

GameplayPauseUI.ExitToMenu:

  if CampaignSessionState.HasActiveCampaignContext
      MenuFlowState.RequestCampaignLevelList(ActiveCampaign)
  else
      MenuFlowState.RequestPlaySubPanel()
  LoadMainMenu()

  HasPendingCampaignRestore
